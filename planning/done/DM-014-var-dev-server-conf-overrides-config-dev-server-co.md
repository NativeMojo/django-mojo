# var/dev_server.conf overrides config/dev_server.conf (fallback if absent)

## What & Why
The testit dev-server host/port is read from `config/dev_server.conf` (committed,
`host`/`port` only). A developer who wants the test server on a different host/port
locally has to edit that tracked file, which risks committing a machine-specific
value.

Add a local override: if `var/dev_server.conf` exists, use it **instead of**
`config/dev_server.conf`; if it does not exist, fall back to
`config/dev_server.conf` (current behavior, unchanged). `var/` is gitignored
(`.gitignore:20`; `testproject/var/` is untracked), so this gives a clean,
uncommitted, per-machine override — mirroring the existing `var/django.conf`
pattern that already lives in `var/`.

**Override semantics (decided with the user): whole-file replace, not per-key
merge.** When `var/dev_server.conf` exists it is the *only* file read; a key it
omits uses the built-in default, NOT config's value.

## Acceptance Criteria
- [ ] When `var/dev_server.conf` exists, all three loaders read `host`/`port` from
      it and ignore `config/dev_server.conf` entirely (whole-file replace).
- [ ] When `var/dev_server.conf` is absent, all three loaders read
      `config/dev_server.conf` exactly as today (no behavior change).
- [ ] When neither file exists, the existing built-in defaults apply — no crash.
- [ ] The uvicorn launcher (`bin/asgi_local`) and both Python readers select the
      **same** effective file, so the server binds and the test client connects on
      the same host/port (no split-brain).
- [ ] No new committed `var/dev_server.conf`; it stays a local-only override.

## Investigation

### What exists — three independent loaders, all hardcoded to config/
The `.conf` is a flat `key=value` file with only `host` and `port`
(`testproject/config/dev_server.conf` = `host=127.0.0.1` / `port=5555`). Three
loaders parse it independently, none aware of the others:

1. `testit/runner.py:237-255` — `get_host()` reads `paths.CONFIG_ROOT /
   "dev_server.conf"`. Defaults `127.0.0.1:8001`; remaps `0.0.0.0`→`127.0.0.1`.
   Silently swallows errors. Feeds the CLI `--host` default.
2. `testit/helpers.py:605-619` — `_read_dev_server_conf()` reads
   `paths.CONFIG_ROOT / 'dev_server.conf'`. Defaults `127.0.0.1:5555`. Already
   guards with `conf.exists()`. Used by `server_settings()` (helpers.py:671).
3. `bin/asgi_local:51-67` — `_read_conf()` (bash) reads
   `"$TESTPROJECT/config/dev_server.conf"` (hardcoded string, NOT via `paths`).
   Defaults `127.0.0.1:5555`. **This is the one that actually binds uvicorn**
   (called from `do_start`/`do_foreground`), so it must change too or the client
   and server disagree.

### What changes
Each loader computes the effective path as "`var/dev_server.conf` if it exists,
else `config/dev_server.conf`" and parses that single file (whole-file replace):
- Python (readers 1 & 2): `paths.VAR_ROOT / 'dev_server.conf'` else
  `paths.CONFIG_ROOT / 'dev_server.conf'`. `VAR_ROOT`/`CONFIG_ROOT` are existing
  globals (`mojo/helpers/paths.py:21,24`).
- Bash (reader 3): `$TESTPROJECT/var/dev_server.conf` else
  `$TESTPROJECT/config/dev_server.conf`.
- Possible DRY for the two Python readers: a tiny `paths` helper (e.g.
  `resolve_conf('dev_server.conf')`) — a /scope design call; bash can't share it.

### Constraints / notes
- Precedent for a var/-hosted `.conf`: `var/django.conf`
  (`testit/helpers.py:669`, generated by `bin/create_testproject`). This introduces
  the first var-*overrides*-config fallback (today `django.conf` is var-only and
  `dev_server.conf` is config-only).
- Pre-existing inconsistency (out of scope, flag only): `runner.py` defaults port
  **8001** while the other two default **5555**. Not introduced here; don't let the
  refactor silently "fix" or entangle it unless /scope decides to.
- `bin/run_tests` drives `bin/asgi_local` then `testit.runner`; both paths must
  agree on the effective file.

## Plan

### Goal
Make all three `dev_server.conf` loaders prefer `var/dev_server.conf` when it
exists (whole-file replace) and otherwise fall back to `config/dev_server.conf`,
giving developers a gitignored, per-machine host/port override.

### Context — what exists
The `.conf` is a flat `key=value` file with only `host` and `port`. The committed
copy `testproject/config/dev_server.conf` is:
```
host=127.0.0.1
port=5555
```
`var/` is gitignored (`.gitignore:20` = `/var`) and `testproject/var/` is untracked,
so a `var/dev_server.conf` is naturally a local-only override — mirroring the
existing `var/django.conf` pattern (`testit/helpers.py:669`, generated by
`bin/create_testproject`). Today `django.conf` is var-only and `dev_server.conf`
is config-only; this item introduces the first var-*overrides*-config fallback.

Path globals live in `mojo/helpers/paths.py`, set at Django startup by
`configure_paths()`:
- `VAR_ROOT = PROJECT_ROOT / "var"` (`mojo/helpers/paths.py:21`)
- `CONFIG_ROOT = PROJECT_ROOT / "config"` (`mojo/helpers/paths.py:24`)

**Three independent loaders**, all hardcoded to config/, none aware of the others:

1. `testit/runner.py:237-255` — `get_host()`. Reads `paths.CONFIG_ROOT /
   "dev_server.conf"` (line 242), parses `host`/`port`, remaps `0.0.0.0`→
   `127.0.0.1`, defaults `127.0.0.1:8001`, swallows errors. Feeds CLI `--host`.
   ```python
   config_path = paths.CONFIG_ROOT / "dev_server.conf"
   with open(config_path, 'r') as file:
       for line in file:
           if line.startswith("host"): ...
   ```
2. `testit/helpers.py:605-619` — `_read_dev_server_conf()`. Reads `paths.CONFIG_ROOT
   / 'dev_server.conf'` (line 609), guards with `conf.exists()`, defaults
   `127.0.0.1:5555`. Used by `server_settings()` (helpers.py:671).
   ```python
   conf = paths.CONFIG_ROOT / 'dev_server.conf'
   if conf.exists():
       for line in conf.read_text().splitlines():
           if '=' in line: k, v = line.split('=', 1) ...
   ```
3. `bin/asgi_local:51-67` — `_read_conf()` (bash). Reads
   `"$TESTPROJECT/config/dev_server.conf"` (line 54), defaults `127.0.0.1:5555`.
   **This is the loader that actually binds uvicorn** (called from `do_start`
   line 110 / `do_foreground` line 166), so it MUST change too or the client and
   server disagree on host/port.
   ```bash
   CONF="$TESTPROJECT/config/dev_server.conf"
   if [ -f "$CONF" ]; then
       while IFS='=' read -r key value; do ... done < "$CONF"
   fi
   ```

uvicorn is started with `--reload --reload-include '*.conf'` and no `--reload-dir`
(`bin/asgi_local:120-125`, `174-179`), so it watches cwd recursively and `var/` is
in the watched tree — `server_settings()` deliberately relies on this
(`testit/helpers.py:640,651`). **Consequence for tests:** writing a real
`var/dev_server.conf` during a run would trigger a uvicorn reload mid-suite, so the
regression test must NOT write into live `var/` — use tmp dirs (see Tests).

Test model/home: `tests/test_helpers/test_runner_config.py` already tests testit
config loaders directly and uses `tempfile.TemporaryDirectory()` for isolation
(`test_config_defaults`, `test_config_partial`) and toggles module globals in a
`try/finally` (`test_agent_report_writes`, lines 155-177). Decorator:
`@th.django_unit_test("...")`; asserts via `th.assert_eq` / `th.assert_true` with
descriptive messages. Import the module under test *inside* the test function
(project convention, see `.claude/rules/testing.md`).

### Changes — what to do
1. `mojo/helpers/paths.py` — add a pure, testable resolver (new function, no type
   hints per `.claude/rules/core.md`):
   ```python
   def resolve_conf(name, var_root=None, config_root=None):
       """Return VAR_ROOT/name if it exists (local override), else CONFIG_ROOT/name.
       Whole-file override — the caller reads the returned file as-is; there is no
       per-key merge. var_root/config_root are injectable for testing."""
       var_root = VAR_ROOT if var_root is None else var_root
       config_root = CONFIG_ROOT if config_root is None else config_root
       var_path = var_root / name
       return var_path if var_path.exists() else config_root / name
   ```
   Placed after `configure_apps()` at module scope; it reads the `VAR_ROOT`/
   `CONFIG_ROOT` globals lazily at call time (they exist once `configure_paths()`
   has run, which is always the case when a loader calls it).
2. `testit/runner.py:242` — replace
   `config_path = paths.CONFIG_ROOT / "dev_server.conf"` with
   `config_path = paths.resolve_conf("dev_server.conf")`. Leave the surrounding
   try/except, `0.0.0.0` remap, and defaults untouched.
3. `testit/helpers.py:609` — replace
   `conf = paths.CONFIG_ROOT / 'dev_server.conf'` with
   `conf = paths.resolve_conf('dev_server.conf')`. Keep the `conf.exists()` guard
   and the rest of the parse.
4. `bin/asgi_local:_read_conf()` (lines 51-67) — mirror the whole-file override in
   bash (no Python helper reachable here). Change the fixed `CONF=` assignment to
   prefer var/ when present:
   ```bash
   CONF="$TESTPROJECT/config/dev_server.conf"
   if [ -f "$TESTPROJECT/var/dev_server.conf" ]; then
       CONF="$TESTPROJECT/var/dev_server.conf"
   fi
   ```
   Leave the existing `if [ -f "$CONF" ]; then ... while IFS='=' ...` parse and the
   `export` lines unchanged.
5. Docs — see Docs section below.

### Design decisions
- **Pure resolver in `paths.py`, not inline `if var.exists()` in each reader.**
  Rationale: (a) DRYs the two Python readers; (b) the `var_root`/`config_root`
  params make it unit-testable against tmp dirs, so the regression test never
  writes into live `var/` — which `--reload-include '*.conf'` would otherwise pick
  up and reload mid-suite. Rejected: duplicating a 2-line check in each reader
  (untestable without touching live var/, and drifts).
- **Whole-file replace, not per-key merge** (decided with the user). When
  `var/dev_server.conf` exists it is the only file read; a key it omits uses the
  built-in default, NOT config's value. Matches the request phrasing "override it,
  else fall back to config" and is simplest (KISS).
- **Bash reader duplicates the logic inline.** A shell script can't import the
  Python helper; a 3-line mirror is the minimal correct change. The Python helper
  and the bash block are the single source of truth for their respective runtimes.
- **Pre-existing default-port mismatch left as-is** — `runner.py` defaults port
  8001 while `helpers.py`/`asgi_local` default 5555. Dormant (the committed
  `config/dev_server.conf` always exists, so defaults never fire in practice) and
  out of scope. Do NOT "fix" or entangle it in this refactor.

### Edge cases & risks
- **Neither file exists** — `resolve_conf` returns the (non-existent) config path;
  each reader's existing missing-file handling then applies (`runner.py` catches
  `FileNotFoundError`→defaults; `helpers.py`'s `conf.exists()` is False→defaults;
  bash `[ -f "$CONF" ]` is False→defaults). No crash. No behavior change vs today.
- **var/ exists but is a partial file** (e.g. only `host`) — whole-file replace
  means the missing key falls to the built-in default, not config's. Intended.
- **Reload risk during tests** — mitigated by testing the pure resolver / injected
  globals against tmp dirs; the suite never writes a real `var/dev_server.conf`.
- **Client/server split-brain** — all three loaders now resolve identically, so
  `bin/asgi_local` (binds), `runner.get_host()` (client base URL), and
  `helpers._read_dev_server_conf()` (server_settings poll) agree. Covered by AC.

### Tests
Add to `tests/test_helpers/test_runner_config.py` (tmp dirs only, never live var/):
- **var wins when both exist** -> `resolve_conf("dev_server.conf", var_root=tmpA,
  config_root=tmpB)` with a file in both returns the `tmpA` path.
- **fallback to config when var absent** -> same call with a file only in `tmpB`
  returns the `tmpB` path.
- **wire-through of the helpers.py reader** -> temporarily set `paths.VAR_ROOT` and
  `paths.CONFIG_ROOT` to tmp dirs (restore both in `finally`, per
  `test_agent_report_writes`), write `var/dev_server.conf` with a distinctive
  `host`/`port`, call `_read_dev_server_conf()`, assert it returns that host/port —
  proving the reader delegates to `resolve_conf` and whole-file replace works
  end-to-end. Every `assert` carries a descriptive message (`.claude/rules/testing.md`).
- Bash `_read_conf` is not testit-testable; it's verified by inspection and the
  "all three loaders agree" acceptance criterion.

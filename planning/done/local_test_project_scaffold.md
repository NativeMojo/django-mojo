## Problem

django-mojo is a framework, not a Django project. To run the test suite in `tests/`,
a host Django project is required. Currently tests can only be run inside an external
project, which is slow and not portable for CI/DevOps.

## Goal

A single script that generates a minimal, disposable Django project inside
`testproject/` (fully `.gitignore`d) that can:
- Start a local ASGI test server (`bin/asgi_local`)
- Run the full test suite via testit (`bin/testit.py`)
- Be regenerated cleanly at any time by re-running the script
- Work both locally and in automated CI pipelines

### `testproject/bin/paths.py`

Sets up sys.path so that `mojo`, `testit`, and `settings` are all importable.
`REPO_ROOT` is the django-mojo root (one level up from testproject/).

```python
import sys
import os
from pathlib import Path

FILE = Path(__file__).resolve()
TESTPROJECT_ROOT = FILE.parent.parent        # django-mojo/testproject/
REPO_ROOT = TESTPROJECT_ROOT.parent          # django-mojo/

# mojo package + testit package both live at repo root
if str(REPO_ROOT) not in sys.path:
    sys.path.insert(0, str(REPO_ROOT))

# settings module lives at testproject/config/
CONFIG_ROOT = TESTPROJECT_ROOT / "config"
if str(CONFIG_ROOT) not in sys.path:
    sys.path.insert(0, str(CONFIG_ROOT))

os.environ.setdefault("DJANGO_SETTINGS_MODULE", "settings")
```

### `testproject/bin/_asgi.py`

Uvicorn entry point. Imports local `paths.py` first to bootstrap sys.path,
then lets mojo create the ASGI application.

```python
import paths  # testproject/bin/paths.py — must be first

from mojo.helpers import paths as mpaths
mpaths.configure_paths(__file__, 2)

from mojo.apps.realtime.routing import create_application
application = create_application()
```

Note on depth: `_asgi.py` is at `testproject/bin/_asgi.py`.
`parents[2]` = `testproject/`, so `PROJECT_ROOT = testproject/../` = repo root.
We want PROJECT_ROOT = `testproject/`, so depth should be 1:
- `parents[1]` of `testproject/bin/_asgi.py` = `testproject/bin/` ... wait that's wrong.

Correction — `configure_paths` is called from `config/settings/__init__.py`, not `_asgi.py`.
`_asgi.py` just imports `paths` (local bootstrap) and calls `create_application()`.
Django settings handle `configure_paths` internally. So `_asgi.py` is simply:

```python
import paths  # testproject/bin/paths.py — bootstraps sys.path

from mojo.apps.realtime.routing import create_application
application = create_application()
```

---

### `testproject/config/settings/__init__.py`

Framework entry point. Calls `configure_paths(__file__, 1)` which resolves:
- `Path(__file__).parents[1]` = `testproject/config/`
- `PROJECT_ROOT = testproject/config/.parent` = `testproject/` ✓
- `VAR_ROOT = testproject/var/` ✓
- `APPS_ROOT = testproject/apps/` ✓
- `CONFIG_ROOT = testproject/config/` ✓

```python
from mojo.helpers import paths, modules, settings

paths.configure_paths(__file__, 1)
modules.load_module_to_globals(paths, globals())
settings.load_settings_profile(globals())   # reads var/profile → imports settings.local
settings.load_settings_config(globals())    # reads var/django.conf → overrides (SECRET_KEY)
```

`load_settings_profile` reads `VAR_ROOT/profile` ("local"), then does
`from settings.local import *` into globals.
`load_settings_config` reads `VAR_ROOT/django.conf` and evals each line into globals
(used to inject the generated SECRET_KEY).

## Bin Scripts (checked into repo)

### `bin/create_testproject`

Bash script. Idempotent — safe to re-run (wipes and recreates testproject/).

Logic:
1. Resolve `REPO_ROOT` (one level up from script)
2. Remove `testproject/` if it exists
3. Create all directories: `testproject/{bin,config/settings/local,apps,var/logs}`
4. Write all files listed above verbatim
5. Generate SECRET_KEY via Python and write to `var/django.conf`
6. Write `var/profile` = `local`
7. Run `python testproject/manage.py migrate --run-syncdb`

SECRET_KEY generation (inline Python):
```bash
SECRET_KEY=$(python3 -c "import secrets, string; print(''.join(secrets.choice(string.ascii_letters + string.digits + '!@#$%^&*') for _ in range(50)))")
echo "SECRET_KEY = '${SECRET_KEY}'" > "$TESTPROJECT/var/django.conf"
```

### `bin/asgi_local`

Bash script.

Logic:
1. Resolve `REPO_ROOT`
2. Confirm `testproject/` exists (exit with message if not — run `create_testproject` first)
3. Check Redis with `redis-cli ping`; if not running, start `redis-server` in background
4. Read `host` and `port` from `testproject/config/dev_server.conf`
5. Set `DJANGO_SETTINGS_MODULE=settings`, `PYTHONUNBUFFERED=1`
6. Run: `uvicorn _asgi:application --app-dir testproject/bin --host $HOST --port $PORT --reload`

### `bin/testit.py`

Python script. Sets up paths then hands off to `testit.runner`.

```python
#!/usr/bin/env python
import os
import sys
from pathlib import Path

SCRIPT_DIR = Path(__file__).resolve().parent   # django-mojo/bin/
REPO_ROOT = SCRIPT_DIR.parent                  # django-mojo/
TESTPROJECT = REPO_ROOT / "testproject"

if not TESTPROJECT.exists():
    print("ERROR: testproject/ not found. Run ./bin/create_testproject first.")
    sys.exit(1)

sys.path.insert(0, str(REPO_ROOT))                      # mojo, testit
sys.path.insert(0, str(TESTPROJECT / "config"))         # settings

os.environ.setdefault("DJANGO_SETTINGS_MODULE", "settings")

from mojo.helpers import paths
paths.configure_paths(str(TESTPROJECT / "config" / "settings" / "__init__.py"), 1)

from testit import runner

if __name__ == "__main__":
    runner.main(runner.setup_parser())
```
